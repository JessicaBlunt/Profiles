utils¶

fpath_temp_calib¶

The file path to the calibration coefficients for the temperature sensors

utils.qc(data)¶
Determines which sensors are not reliable from a given set. Be sure
to only include like sensors (not both temperature inside and outside
the CO2 sensor) in Data.
Parameters:data (list<Quantity>) – a list containing one list for each sensor in the ensemble, i.e. all external RH sensors
Return type:list of length len(data)
Returns:list containing 1 in the position of each “good” sensor, 2 in the position of each sensor flagged for bias, and 3 in the position of each sensor flagged for response time.
utils.temp_calib(temp, coefs)¶

Converts voltage to temperatures using the given coefficients.

Parameters:
  • temp (list<Quantity>) – voltages recorded by temperature sensors
  • coefs (list<Quantity>) – coefficients to convert voltage to temperature. The coefficients should be determined seperately for each sensor
Return type:

list<Quantity>
Returns:

list of temperatures in K
utils.correct_alt_pres(gps, pres)¶

Uses pressure data to correct altitude data.

Parameters:
  • gps (tuple) – GPS data as (lat, lon, alt_MSL, time: Datetime)
  • pres (tuple) – barometer data as (pres, time: ms)
Return type:

tuple
Returns:

gps with corrected altitudes
utils.rotate(u, v, w, yaw, pitch, roll)¶

Calculate the value of u, v, and w after a specified axis rotation

Parameters:
  • u (list<number>) – U component of the wind
  • v (list<number>) – V component of the wind
  • w (list<number>) – W component of the wind
  • yaw (list<number>) – Rotation about the Z axis
  • pitch (list<number>) – Rotation about the X axis
  • roll (list<number>) – Rotation about the Y axis
Return type:

np.array
Returns:

3D array of the new U, V, and W fields after the rotation
